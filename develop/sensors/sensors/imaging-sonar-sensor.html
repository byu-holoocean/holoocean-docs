Imaging Sonar

Sometimes referred to as a forward-looking sonar, imaging sonar is a type of multi-beam sonar that projects the intensity onto the 2D time of return-azimuth plane. This allows for a 2D image to be created of the area in front of the sonar. This is useful for detecting objects in front of the sonar, and is often used in AUVs to detect obstacles in front of them.

Holoocean has three different imaging sonar sensor implementations:

  • Octree Imaging Sonar

  • Raycast Imaging Sonar

  • GPU Imaging Sonar

These implementations have different tradeoffs in accuracy, speed, and memory usage.

Octree Imaging Sonar

Simulates an imaging multi-beam sonar with octrees. It projects the intensity onto the 2D time of return-azimuth plane.

See ImagingSonar for the python API.

Example sensor definition:

{
    "sensor_type": "ImagingSonar",
    "socket": "SonarSocket",
    "Hz": 2,
    "configuration": {
        "Azimuth": 120,
        "Elevation": 20,
        "RangeMin": 0.1,
        "RangeMax": 10,
        "RangeBins": 512,
        "AzimuthBins": 512,
        "AddSigma": 0.15,
        "MultSigma": 0.2,
        "MultiPath": True,
        "ClusterSize": 5,
        "ScaleNoise": True,
        "AzimuthStreaks": -1,
        "RangeSigma": 0.1,
        "ShowWarning": True,
        "InitOctreeRange": 50,
        "ViewRegion": False,
        "ViewOctree" -10,
        "WaterDensity": 997,
        "WaterSoundSpeed": 1480,
        "UseApprox": True,
    }
}

Note that the Azimuth streaks parameter is used to control whether the simulator will add or remove streaks along the azimuth to the return, as those are common artifacts in sonar images.

Raycast Imaging Sonar

Simulates an imaging multi-beam sonar using raycasting. It projects the intensity onto the 2D time of return-azimuth plane.

See RaycastImagingSonar for the python API.

Example sensor definition:

{
    "sensor_type": "RaycastImagingSonar",
    "socket": "SonarSocket",
    "Hz": 10,
    "configuration": {
        "RangeBins": 512,
        "AzimuthBins": 512,
        "RangeMin": 0.1,
        "RangeMax": 10,
        "Elevation": 20,
        "Azimuth": 120,
        "MultSigma": 0.0,
        "AddSigma": 0.0,
        "MultiPath": False,
        "ViewRegion": False,
        "WaterDensity": 997,
        "WaterSpeedSound": 1480,
        "IgnorePlants": False
    }
}

GPU Imaging Sonar

Simulates an imaging multi-beam sonar using GPU rendering. It projects the intensity onto the 2D time of return-azimuth plane. Utilizes the GPU rendering pipeline, for faster simulation speed.

See GPUImagingSonar for the python API.

Example sensor definition:

{
    "sensor_type": "GPUImagingSonar",
    "socket": "SonarSocket",
    "Hz": 10,
    "configuration": {
        "RangeBins": 512,
        "AzimuthBins": 512,
        "RangeMin": 0.1,
        "RangeMax": 10,
        "Elevation": 20,
        "Azimuth": 120,
        "MultSigma": 0.0,
        "AddSigma": 0.0,
        "MultiPath": False,
        "ViewRegion": False,
        "WaterDensity": 997,
        "WaterSpeedSound": 1480,
        "IgnorePlants": False
    }
}

Resolution Calculation for Raycast and GPU Imaging Sonars

The raycast and GPU imaging sonars require sufficient angular sampling to accurately populate the desired range bins. This section provides a method for estimating the required angular resolution based on the sonar field of view and range-bin resolution.

Angular Step Size Calculation

From the Azimuth Field of View

A conservative estimate of the required angular step size can be computed as:

\[\theta_{azi} = \frac{\Theta}{2} - \cos^{-1}\!\left( \frac{R_{\max}}{R_{\max}-R_{\mathrm{res}}} \cos\!\left(\frac{\Theta}{2}\right) \right)\]

where:

  • \(\Theta\) is the total azimuth angle.

  • \(R_{\max}\) is the maximum sonar range.

  • \(R_{\mathrm{res}}\) is the range resolution.

Note

The range resolution can be computed from the number of range bins:

\[R_{\mathrm{res}} = \frac{R_{\max} - R_{\min}}{R_{bins}}\]

This formulation assumes a worst-case geometry in which the seafloor or target intersects the edge of the sonar field of view at maximum range, as illustrated below:

From the Elevation Field of View

A conservative estimate of the required angular step size can be computed as:

\[\theta_{elev} = \frac{\alpha}{2} - \cos^{-1}\!\left( \frac{R_{\max}}{R_{\max}-R_{\mathrm{res}}} \cos\!\left(\frac{\alpha}{2}\right) \right)\]

where:

  • \(\alpha\) is the total elevation angle.

  • \(R_{\max}\) is the maximum sonar range.

  • \(R_{\mathrm{res}}\) is the range resolution.

This formulation assumes a worst-case geometry in which the seafloor or target intersects the edge of the sonar field of view at maximum range, as illustrated below:

Choosing Angular Step Size

The angular step size must satisfy both the azimuth and elevation constraints. Therefore, the more restrictive (smaller) step size should be used:

\[\theta = min(\theta_{azi}, \theta_{elev})\]

The required number of samples is then:

\[ \begin{align}\begin{aligned}N_{azi} = \frac{\Theta}{\theta}\\N_{elev} = \frac{\alpha}{\theta}\end{aligned}\end{align} \]

This calculation represents a worst-case estimate and guarantees sufficient resolution across the full field of view.

How To Input Resolution Into HoloOcean

For Raycast Imaging Sonar

There are multiple parameters that you can set to change the resolution and speed of the raycast sonar. The options are listed in the code block below:

Note

If you don’t manually set your resolution, the resolution will be calculated based off of your other parameters using the above equations.

{
    "sensor_type": "RaycastImagingSonar",
    "socket": "SonarSocket",
    "Hz": 10,
    "configuration": {
        "AzimuthRayCount":   <N_azi>,    # set one or the other
        # "AzimuthRayRes":   <theta>,
        "ElevationRayCount": <N_elev>,   # set one or the other
        # "ElevationRayRes": <theta>,
    }
}

where \(N_{azi}\), \(N_{elev}\), and \(\theta\) are computed using the equations above.

For GPU Imaging Sonar

There is only one parameter that you can set to change the resolution and speed of the GPU sonar. This parameter, ResolutionX, is computed as the maximum of the azimuth or elevation ray counts times 2.

\[N_{x} = 2 * max(N_{azi}, N_{elev})\]

The parameter is set in the configuration dictionary as shown below:

{
    "sensor_type": "GPUImagingSonar",
    "socket": "SonarSocket",
    "Hz": 10,
    "configuration": {
        "ResolutionX": <N_x>,
    }

}
